## Interfaces (pinned §2.15 — implement exactly; adapt names only where the doc conflicts with itself)

```rust
pub struct LayoutParams { pub spring_k: f32, pub repulsion: f32, pub damping: f32, pub settle_eps: f32 }
impl Default for LayoutParams { /* tuned here, then frozen */ }
pub struct ForceLayout { /* nodes, springs, spatial grid, velocities */ }
impl ForceLayout {
    /// `pinned` = positions restored from the cache; they participate but keep spatial identity
    /// (they move like everyone else AFTER load; "pinned" means initial placement, not frozen).
    pub fn new(node_ids: &[NoteId], edges: &[(NoteId, NoteId)], pinned: &HashMap<NoteId, Vec2>, params: LayoutParams) -> Self;
    pub fn step(&mut self, dt: f32) -> f32;      // returns max displacement
    pub fn is_settled(&self) -> bool;            // max displacement < settle_eps → freeze
    pub fn positions(&self) -> &HashMap<NoteId, Vec2>;
    pub fn add_node(&mut self, id: NoteId, edges: &[NoteId]);  // seed near neighbor centroid
}
```
Complexity requirement: repulsion via uniform spatial grid (cell ≈ 2× node spacing), O(n·k)/step. Bench `#[test]` (release-gated + serial, mirroring jd-core's perf.rs conventions exactly): one `step()` at 20k nodes / 40k edges < 16 ms.

### Task 1: jd-core — ForceLayout + MapCache + bench

**Files:** create `crates/jd-core/src/maplayout.rs` (+ lib.rs line), bench per conventions.

- ForceLayout per the pinned interface. Deterministic given identical inputs (no RNG except initial placement of UNPINNED nodes — seed a jd-core xorshift from a FIXED constant so layouts are reproducible; document). Springs along edges (rest length constant, pin ~120.0), grid-bucketed repulsion, velocity damping, `step` returns max displacement; `is_settled` when max displacement < settle_eps.
- Initial placement: pinned nodes at their cached pos; unpinned nodes near the centroid of their pinned/placed neighbors + small deterministic jitter, or on a spiral for orphans/no-neighbor nodes.
- `add_node`: neighbor-centroid seed + jitter; marks layout unsettled.
- **MapCache**: `.junkdrawer/map.jd` — line format mirroring session.jd (`jd-map 1` header; `node = <ulid> <x> <y>` lines). `load(vault) -> HashMap<NoteId, Vec2>` (missing/corrupt → empty, disposable by design); `save(vault, positions) -> Result<(), IoError>` atomic. MIRROR session.rs's structure/idioms exactly (read it first).
- Unit tests: two linked nodes end closer than two unlinked (spring sanity); a triangle settles (is_settled true within N steps, positions stable after); pinned nodes keep exact initial positions at step 0 and drift smoothly (no teleport: first-step displacement bounded); determinism (same inputs → same positions after k steps); add_node seeds within radius of neighbor centroid; cache round-trip byte test + corrupt-file → empty.
- Bench: 20k nodes / 40k edges (xorshift-generated, reproducible), one `step()` < 16 ms, release-gated (`#[cfg_attr(debug_assertions, ignore)]`), single-threaded conventions per perf.rs. Also assert settle happens (< some steps) at 1k nodes as a non-gated sanity.
